checkin_file.xml

This SOAP request allows you to check-in the new version of a file that was previously checked-out from the xincoDMS repository.

The response contains the XincoCoreData object representing the new state of the file which should be the same as before but with the status number set to 1.

Important note: This template can only be used to check-in FILE objects. It is not applicable for texts, URLs or contacts!

Remember that checking-in a data object is done in multiple steps. Be sure to follow this process:

  1. Check-in the file using this SOAP request
  2. Create a log entry for the check-in using create_log.xml
  3. Upload the new binary data using upload_file.xml

Basic parameters

The text content of the following element nodes has to be adjusted:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/designation Name of the file you are checking-in
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_node_id Internal ID of the parent folder containing the file (e.g. 1001)
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/id Internal ID of the file to be checked-in (e.g. 1001)
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in1/username Username of the user performing the operation
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in1/userpassword Password of the user performing the operation

Setting the meta-information

Even if you do not want to modify the meta-information of file you have to specify them in this request since xincoDMS needs to know the new filesize. You can copy all values from a previous version except the Size value which must be set to the new filesize (in bytes). You may query each XincoAddAttribute object by its <attribute_id> element using these conditional XPath expressions:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '1'] Name of the file

Set the text content of the <attrib_varchar> child to the designation of the file.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '2'] Size of the file (in bytes)

Set the text content of the <attrib_unsignedint> child to the filesize in bytes.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '3'] Checksum of the file

This feature is currently not supported by xincoDMS.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '4'] Revision model of the file (keep old versions)

Set the text content of the <attrib_unsignedint> child to 1 if you want to enable the revision model for this file (keep old versions) or to 0 to disable this feature.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '5'] Archiving model of the file

Set the text content of the <attrib_unsignedint> child to 1 if you want to enable archiving for this file or 0 to disable it.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '6'] Last archiving date of the file

Set the text content of the <attrib_datetime> child to the last date the file was archived or skip it if you do not use archiving. Date syntax: YYYY-MM-DDTHH:MM:SS.DDDZ
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '7'] Archiving interval of the file (in days)

Set the text content of the <attrib_unsignedint> child to a value defining the number of days between archiving operations or leave it out if you do not use archiving.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '8'] Archiving location of the file

Set the text content of the <attrib_text> child to the desired archiving location or leave just skip this item if you do not use archiving.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '9'] Description of the file

Set the text content of the <attrib_varchar> child to the description of the file.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '10'] Keyword 1 of the file

Set the text content of the <attrib_varchar> child to the first keyword of the file.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '11'] Keyword 2 of the file

Set the text content of the <attrib_varchar> child to the second keyword of the file.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_add_attributes/item[attribute_id/text() = '12'] Keyword 3 of the file

Set the text content of the <attrib_varchar> child to the third keyword of the file.

Important note: The text content of the <xinco_core_data_id> child element in each <item> node must be set to the internal ID of the file you are checking-in.

Preparing the log entry for the operation

Although the log entry for the check-in operation is created in the next step you already have to fill in the log information here. So you have to modify the text content of the following elements:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_logs/item/op_description Log description for this file operation. Here you can provide a brief description of what is new in this version.
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_logs/item/version/version_high Major version of the new file state
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_logs/item/version/version_low Revision of the new file state
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_logs/item/version/version_mid Minor version of the new file state
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_logs/item/version/version_postfix Version postfix of the new file state (e.g. Beta)
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_logs/item/xinco_core_data_id Internal ID of the file you are checking-in (e.g. 1001)
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_logs/item/xinco_core_user_id Internal ID of the user performing this operation. This can be obtained via the get_user_info.xml request.

Language settings

If you want to set the language of this file object you have to adjust the following nodes as well. By default the language is pre-configured to be "unknown".

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_language/designation Name of the desired language

Predefined values in the xincoDMS basic installation:
unknown, English, German, French, Italian, Spanish, Russian
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_language/id Internal ID of the desired language

Predefined values in the xincoDMS basic installation:
1 = unknown
2 = English
3 = German
4 = French
5 = Italian
6 = Spanish
7 = Russian
/soapenv:Envelope/soapenv:Body/ns1:doXincoCoreDataCheckin/in0/xinco_core_language/sign Abbreviation of the desired language

Predefined values in the xincoDMS basic installation:
n/a = unknown
en = English
de = German
fr = French
it = Italian
es = Spanish
ru = Russian

Hint: Each of the XPath queries above will return an element node. To modify its content you will have to go down to its text node which is typically the first child. Be prepared that some elements might not have a text node by default. You may have to create a child text node first.
