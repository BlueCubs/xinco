create_log.xml

This SOAP request provides a fundamental feature for many operations with data objects inside the xincoDMS repository. It enables you to create log entries which are required for any operation on data objects.

The response contains a XincoCoreLog object with its ID and the complete information you provided.

Log entries are an essential component of xincoDMS. You must create such entries for each and every action you perform on data objects, including upload, check-in, check-out, modification of meta-information etc. Downloading an object for reading does not require any logging.

Basic parameters

The text content of the following element nodes has to be adjusted:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in0/op_code Operation code of the operation the log entry is created for.

Predefined values:
1 = Creation (e.g. first upload of a file to the repository)
2 = Modification (e.g. changing the meta-information or moving a file)
3 = Check-out (downloading for writing purpose)
4 = Undo Check-out (revert check-out)
5 = Check-in (e.g. uploading a new version of a file)
6 = Publish (make data object available via web)
7 = Lock (deny any access to the object)
8 = Archive (file has been archived)
9 = Comment (comment has been created for this object)
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in0/op_description Log description of the operation where you should describe what the action associated with this log entry did to the data object.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in0/version/version_high Major version of the file after this operation
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in0/version/version_low Revision of the file after this operation
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in0/version/version_mid Minor version of the file after this operation
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in0/version/version_postfix Version postfix of the file after this operation (e.g. Beta)
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in0/xinco_core_data_id Internal ID of the file that was processed by the operation
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in0/xinco_core_user_id Internal ID of the user performing the operation. This can be obtained via the get_user_info.xml request.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in1/username Username of the user performing the operation
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreLog/in1/userpassword Password of the user performing the operation

Hint: Each of the XPath queries above will return an element node. To modify its content you will have to go down to its text node which is typically the first child. Be prepared that some elements might not have a text node by default. You may have to create a child text node first.
