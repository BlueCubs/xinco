Frequently Asked Questions

In this part of the documentation some frequently questions should be addressed. Please excuse me if your question is not listed here. I will not go into detail with certain programming languages. So this is where your own creativity is challenged.

What do I need for the communication with xincoDMS?

Actually you only need the following components to achieve a successful communication betwenn any client application and the xincoDMS server:

  • The SOAP templates in this package
  • A programming language supporting XML (and XPath) and HTTP communication
  • Some experience with XML and HTTP or a powerful coffee machine in case you are new to the topic ;)

How do I send a request to xincoDMS?

First of all you should load the appropriate SOAP template from this package. Then use an XML parser (with XPath) to obtain the nodes described in the usage documentation and fill in your own values. Finally send the modified XML document via HTTP to the xincoDMS web service.

Please remember that some operations like uploading a new file are done in multiple steps. Be sure to comply with the step-by-step procedure or your action will fail.

My requests take unashamedly long to complete. What can I do?

If this behaviour occurs during the upload or download of binary data it might be quite normal if your files have a certain size. Due to the Base64 encoding of the data the volume gets inflated by additional 30%. Remember that most private internet connections are asymmetric in up- and downstream. So the upload of data to the server might take much longer than downloading.

If you experience such a phenomenon in a search operation you have probably sent a query which returns many results. So the server response can be quite large. Make sure you specify your search in many details to reduce the number of matches.

How do I delete an object from the repository?

Indeed this does not go the obvious way since there is no "Delete" request. If you want to remove a data object or folder from the repository you have to move it to the "Trash" folder. Use the create_file.xml request for files or the create_folder.xml request for folders and always set the ID of the parent folder to 2 - this is the pre-definied value for the trash.

Note that the files and folders are not deleted automatically. The administrator of your xincoDMS server has to empty the trash from time to time. This is done in the XincoAdmin interface.

Why are there no templates for locking, commenting and publishing data?

These operations (and some more) were not required in the context of the project which lead to these SOAP templates. You can build these files on your own by using the SOAPMonitor applet available for Axis.
