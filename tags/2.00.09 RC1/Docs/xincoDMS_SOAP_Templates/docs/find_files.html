find_files.xml

This SOAP request allows you to search for certain data objects in the xincoDMS repository.

The response contains a vector of search results including the XincoCoreData object and all XincoCoreNode objects representing the data's path in the repository.

Be careful: The response message can grow very large if there are lots of search results or the data objects are located in a deep hierarchy. So be sure to use a very precise search query to reduce the number of matches.

Basic parameters

The text content of the following element nodes has to be adjusted:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:findXincoCoreData/in0 Search query defining the desired data objects.
Example: Keyword_1:foo AND Description:bar

The syntax is the same as in XincoExplorer's search function.
/soapenv:Envelope/soapenv:Body/ns1:findXincoCoreData/in2/username Username of the user performing the operation
/soapenv:Envelope/soapenv:Body/ns1:findXincoCoreData/in2/userpassword Password of the user performing the operation

Searching for a specific language

If you want to search for objects in a certain language only you also have to specify the desired language. Normally the search will return objects of any language.

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:findXincoCoreData/in1/designation Name of the desired language

Predefined values in the xincoDMS basic installation:
unknown, English, German, French, Italian, Spanish, Russian
/soapenv:Envelope/soapenv:Body/ns1:findXincoCoreData/in1/id Internal ID of the desired language

Predefined values in the xincoDMS basic installation:
1 = unknown
2 = English
3 = German
4 = French
5 = Italian
6 = Spanish
7 = Russian
/soapenv:Envelope/soapenv:Body/ns1:findXincoCoreData/in1/sign Abbreviation of the desired language

Predefined values in the xincoDMS basic installation:
n/a = unknown
en = English
de = German
fr = French
it = Italian
es = Spanish
ru = Russian

Hint: Each of the XPath queries above will return an element node. To modify its content you will have to go down to its text node which is typically the first child.
