SOAP templates for xincoDMS documentation and user manual

This documentation briefly describes how to use the SOAP templates for communication with the xincoDMS server. It will not illustrate large examples in a particular programming language but rather give an overview of how to integrate the SOAP templates in a custom application environment.

Table of contents

  1. Interaction with the xincoDMS server
  2. Concept of the SOAP templates
  3. Usage of the SOAP templates
    • checkin_file.xml (Check-in a new version of a file object)
    • checkout_file.xml (Download a file from the repository for writing)
    • create_ace.xml (Grant permissions on folders or data objects)
    • create_file.xml (Create, modify or move a file in the repository)
    • create_folder.xml (Create or rename a folder in the repository)
    • create_log.xml (Create a log entry for any data operation)
    • download_file.xml (Download any version of a file for reading only)
    • find_files.xml (Search for data objects)
    • get_file_info.xml (Get detailed information on data objects by ID)
    • get_folder_content.xml (Query information and subordinated elements of any folder by ID)
    • get_user_info.xml (Retrieve detailed user information by username and password)
    • remove_ace.xml (Revoke permissions on folders or data objects)
    • undo_checkout_file.xml (Revert a previous check-out)
    • upload_file.xml (Transfer binary data to the repository)
  4. Frequently Asked Questions
