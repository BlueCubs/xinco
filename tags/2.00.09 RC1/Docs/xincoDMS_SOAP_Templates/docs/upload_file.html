upload_file.xml

This SOAP request marks the final step in the process of file upload or check-in. It allows you to transfer the binary data of the file to the xincoDMS repository.

The response contains a value which should be equal to the filesize (in bytes) on successful transfer.

Important note: The binary data is transfered in Base64 encoding. You have to perform the appropriate transformation manually.

Remember that uploading or checking-in a data object is done in multiple steps. Be sure to follow this process:

  1. Create the file on the server using create_file.xml or check-in the file using checkin_file.xml
  2. Create a log entry for the creation/check-in using create_log.xml
  3. Upload the binary data using this SOAP request

Basic parameters

The text content of the following element nodes has to be adjusted:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/designation Name of the file you are uploading
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/changerID Internal ID of the user performing the file upload (e.g. 1001)
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_node_id Internal ID of the parent folder containing the file (e.g. 1001)
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/id Internal ID of the file to be uploaded. This can be obtained by the create_file.xml request.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in1 Base64-encoded binary data of the file
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in2/username Username of the user performing the operation
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in2/userpassword Password of the user performing the operation

Setting the meta-information

Although already configured in the previous steps of the upload/check-in process you have to fill in the additional attributes (meta-information) in this request as well. It is highly recommended to reuse the values from the previous creation or check-in request. You may query each XincoAddAttribute object by its <attribute_id> element using these conditional XPath expressions:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '1'] Name of the file

Set the text content of the <attrib_varchar> child to the designation of the file.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '2'] Size of the file (in bytes)

Set the text content of the <attrib_unsignedint> child to the filesize in bytes.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '3'] Checksum of the file

This feature is currently not supported by xincoDMS.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '4'] Revision model of the file (keep old versions)

Set the text content of the <attrib_unsignedint> child to 1 if you want to enable the revision model for this file (keep old versions) or to 0 to disable this feature.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '5'] Archiving model of the file

Set the text content of the <attrib_unsignedint> child to 1 if you want to enable archiving for this file or 0 to disable it.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '6'] Last archiving date of the file

Set the text content of the <attrib_datetime> child to the last date the file was archived or skip it if you do not use archiving. Date syntax: YYYY-MM-DDTHH:MM:SS.DDDZ
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '7'] Archiving interval of the file (in days)

Set the text content of the <attrib_unsignedint> child to a value defining the number of days between archiving operations or leave it out if you do not use archiving.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '8'] Archiving location of the file

Set the text content of the <attrib_text> child to the desired archiving location or leave just skip this item if you do not use archiving.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '9'] Description of the file

Set the text content of the <attrib_varchar> child to the description of the file.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '10'] Keyword 1 of the file

Set the text content of the <attrib_varchar> child to the first keyword of the file.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '11'] Keyword 2 of the file

Set the text content of the <attrib_varchar> child to the second keyword of the file.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_add_attributes/item[attribute_id/text() = '12'] Keyword 3 of the file

Set the text content of the <attrib_varchar> child to the third keyword of the file.

Important note: The text content of the <xinco_core_data_id> child element in each <item> node must be set to the internal ID of the file you are uploading.

Reusing the log entry for the operation

The previously created log entry for the file creation/check-in must be put into this template again. Therefore you have to modify the text content of the following elements:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/id Internal ID of the log entry created for this file operation (e.g. 1001)
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/op_code Operation code for this file operation. Set to 1 for the creation of a new file or to 5 for check-in.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/changerID Internal ID of the user performing this operation. This can be obtained via the get_user_info.xml request.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/op_description Log description for this file operation. Here you can provide a brief description of what you did to the file.
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/version/version_high Major version of the file at this state
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/version/version_low Revision of the file at this state
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/version/version_mid Minor version of the file at this state
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/version/version_postfix Version postfix of the file at this state (e.g. Beta)
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/xinco_core_data_id Internal ID of the file you are uploading (e.g. 1001)
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_logs/item/xinco_core_user_id Internal ID of the user performing this operation. This can be obtained via the get_user_info.xml request.

Language settings

If you want to set the language of this file object you have to adjust the following nodes as well. By default the language is pre-configured to be "unknown".

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_language/designation Name of the desired language

Predefined values in the xincoDMS basic installation:
unknown, English, German, French, Italian, Spanish, Russian
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_language/id Internal ID of the desired language

Predefined values in the xincoDMS basic installation:
1 = unknown
2 = English
3 = German
4 = French
5 = Italian
6 = Spanish
7 = Russian
/soapenv:Envelope/soapenv:Body/ns1:uploadXincoCoreData/in0/xinco_core_language/sign Abbreviation of the desired language

Predefined values in the xincoDMS basic installation:
n/a = unknown
en = English
de = German
fr = French
it = Italian
es = Spanish
ru = Russian

Hint: Each of the XPath queries above will return an element node. To modify its content you will have to go down to its text node which is typically the first child. Be prepared that some elements might not have a text node by default. You may have to create a child text node first.
