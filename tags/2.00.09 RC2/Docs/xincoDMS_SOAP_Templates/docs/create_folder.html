create_folder.xml

This SOAP request allows you to create a new folder in the xincoDMS repository or it can be used to rename an existing folder by submitting its ID and the new designation.

The response contains the new XincoCoreNode object with its ID, designation, parent ID and all subordinated elements (if any).

Basic parameters

The text content of the following element nodes has to be adjusted:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreNode/in0/designation New Name of the folder that is created/renamed
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreNode/in0/xinco_core_node_id Internal ID of the parent folder that should contain the created/renamed folder
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreNode/in0/id Internal ID of the folder that is renamed or 0 to create the folder
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreNode/in1/username Username of the user performing the operation
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreNode/in1/userpassword Password of the user performing the operation

Language settings

If you want to use language settings you have to adjust the following nodes as well. Otherwise the language is automatically set to "unknown" (pre-configured in the template).

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreNode/in0/xinco_core_language/designation Name of the desired language

Predefined values in the xincoDMS basic installation:
unknown, English, German, French, Italian, Spanish, Russian
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreNode/in0/xinco_core_language/id Internal ID of the desired language

Predefined values in the xincoDMS basic installation:
1 = unknown
2 = English
3 = German
4 = French
5 = Italian
6 = Spanish
7 = Russian
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreNode/in0/xinco_core_language/sign Abbreviation of the desired language

Predefined values in the xincoDMS basic installation:
n/a = unknown
en = English
de = German
fr = French
it = Italian
es = Spanish
ru = Russian

Hint: Each of the XPath queries above will return an element node. To modify its content you will have to go down to its text node which is typically the first child.
