create_ace.xml

This SOAP request allows you to add permissions for users or groups on certain folders or data objects.

The response contains the XincoCoreACE object that has been created.

Basic parameters

The text content of the following element nodes has to be adjusted:

XPath queryValue
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in0/admin_permission Set "true" to enable admin permission for the user/group on the object or "false" to disable it.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in0/execute_permission Set "true" to enable execute permission for the user/group on the object or "false" to disable it.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in0/xinco_core_data_id Internal ID of the data object you are creating this ACE for or 0 if you do not want to create this permission for a data object.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in0/xinco_core_group_id Internal ID of the user group the permission is granted to or 0 if you do not want to create this permission for a group.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in0/xinco_core_node_id Internal ID of the folder you are creating this ACE for or 0 if you do not want to create this permission for a folder.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in0/xinco_core_user_id Internal ID of the user the permission is granted to or 0 if you do not want to create this permission for a user.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in0/read_permission Set "true" to enable read permission for the user/group on the object or "false" to disable it.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in0/write_permission Set "true" to enable write permission for the user/group on the object or "false" to disable it.
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in1/username Username of the user performing the operation
/soapenv:Envelope/soapenv:Body/ns1:setXincoCoreACE/in1/userpassword Password of the user performing the operation

Hint: Each of the XPath queries above will return an element node. To modify its content you will have to go down to its text node which is typically the first child.
